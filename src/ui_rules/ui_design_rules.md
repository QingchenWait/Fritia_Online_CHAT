这个文档里，是设计当前项目聊天软件现代 UI 的基本准则。对于已经实现的业务功能，需要调整布局、组件结构、CSS 样式、响应式适配和必要的前端交互状态，以实现本文档中所描述的 UI 效果。

必须同时设计并规划 2 套 UI，适配移动端竖屏和桌面端横屏。UI 应该存在各种切换动画和效果。必须从互联网上下载合适的图标和其他图像 (放在 ./src/_logo 目录中)，必须更倾向于使用互联网上下载的美术资源 (不要让 codex 自己画，而是优先从网上下载优质美术资源，包括 logo、图片素材等)。 



## 重要响应式约束：
所有尺寸描述均为视觉参考，不代表硬编码 px。请使用 rem、clamp()、百分比、minmax()、Flex/Grid 和媒体查询实现真正响应式布局，确保 1920x1080、2560x1440、4K、浏览器 125%/150% 缩放、以及各种分辨率的移动端竖屏下，都不会出现控件过大、过小、重叠或横向溢出。



## 目标效果：
整体风格为现代化 Soft UI / 轻量玻璃拟态 / Apple-like 桌面应用风格。界面应干净、明亮、圆润、轻盈，主色为蓝紫渐变，背景为浅灰白冷色调。所有窗口、卡片、弹窗、下拉菜单都应具有大圆角、柔和阴影、细边框、轻微半透明背景和充足留白。


## 重要：必须遵守的 UI 图标要求
对于软件里的按钮，必须尽可能多地从网络上下载并使用 SVG 或 PNG 图标作为按钮元素。如果按钮宽度足够，那么就使用“文字+图标”的形式构造按钮。如果按钮较小，则只使用图标构造按钮。除非用户在提示词或给出的设计图中，明确说明按钮不包含图标元素，否则应尽可能地使用外部下载的图标资源，不能够大量使用纯文字图标，禁止使用自行绘制 SVG 的方式代替外部图标资源。图标资源均应该放置在 `./src/_logo` 目录中。

可参考项目中 `./src\ui_rules\UI_Design.png` 视觉概念设计图。



## 核心设计语言：
1. 背景使用浅色渐变或极淡蓝紫灰背景，例如 #F5F7FC / #F8FAFF。
2. 主色使用蓝紫色，例如 #4F63FF、#635BFF、#6C5CFF，可用于当前选中状态、发送按钮、主操作按钮、浮动按钮。
3. 边框使用极浅灰蓝色，例如 rgba(120, 140, 180, 0.18)。
4. 阴影应柔和，不要厚重，例如 0 16px 40px rgba(40, 60, 120, 0.10)。
5. 圆角统一较大，但不要写死固定像素，使用 rem 或 clamp：
   - 主窗口：大圆角
   - 卡片/弹窗：中大圆角
   - 输入框/按钮：中等圆角
   - 头像保持圆形。
6. 字体保持清晰现代，优先使用系统字体：
   - system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif。
7. 图标线条保持简洁，避免复杂装饰。
8. 整体不要使用强烈分割线，改为细边框、浅阴影和留白区分层级。



## 需要保留并美化的已有元素：
- 桌面端聊天界面的左侧竖向功能栏，包括聊天、角色、添加、关系图/知识图谱、设置等图标。
- 顶部用户区域，包括头像、用户名“分析员”、菜单按钮等。
- 顶部用户区域下方的搜索框：“搜索角色、群聊或消息”。
- “会话 / 角色 / 群聊”分段按钮。
- 角色列表，包括各个聊天角色的头像、名称、消息、时间等。
- 主聊天区域，包括顶部角色信息栏、电话/视频/资料等按钮、双方头像、昵称、聊天气泡、时间。
- 底部输入栏，包括表情、图片、附件、语音、输入 placeholder、发送按钮。
- 设置内部窗口。
- 点击按钮后的下拉菜单，例如更多菜单、添加菜单、模型选择菜单。
- 人物资料卡片，包括角色头像、名称、标签、简介、统计信息、发起会话、查看详情等按钮。



桌面端布局要求：
1. 主聊天界面为横屏宽布局，整体像一个独立桌面应用窗口。
2. 左侧竖向功能栏使用响应式窄栏，例如 clamp(3.75rem, 4.5vw, 5rem)，使用白色或半透明背景，图标纵向排列，当前选中项使用蓝紫色高亮背景或左侧蓝紫指示条。
3. 会话列表栏使用响应式宽度，例如 clamp(16rem, 22vw, 22rem)，与主聊天区域之间使用极浅边框分隔。
4. 主聊天区占据剩余空间，使用 minmax(0, 1fr)，顶部栏固定，底部输入栏固定，中间聊天记录区域留出大量空白。
5. 聊天气泡：
   - AI 消息靠左，白色气泡，轻微阴影或细边框。
   - 用户消息靠右，浅蓝或蓝紫渐变气泡，文字清晰可读。
   - 气泡圆角较大，但靠近头像的一侧可以稍微减少圆角。
6. 底部输入栏应设计为悬浮胶囊卡片，距离窗口边缘使用响应式留白，内部图标、输入框和发送按钮横向排列。
7. 发送按钮使用蓝紫渐变，带纸飞机图标，圆角明显。
8. 设置窗口应表现为桌面端内部弹窗或内嵌窗口，不要跳转成完全不同页面。窗口居中或作为独立面板显示，带大圆角、阴影和右上角关闭按钮。
9. 下拉菜单应是悬浮卡片样式，带白色背景、圆角、阴影、细边框。菜单项左侧有图标，危险操作如“清空聊天记录 / 删除聊天”使用红色。
10. 人物资料卡片应是独立悬浮卡片，顶部可有角色头像或横幅区域，名称旁可有小徽章，底部放置主要按钮。



移动端布局要求：
1. 需要适配竖屏手机界面，不要把列表界面和聊天界面重叠在同一个手机屏幕里。
2. 移动端应拆分为独立页面状态：
   - 会话/好友列表页
   - 单人聊天页
   - 侧边菜单页
   - 设置页
3. 列表页顶部显示“分析员”头像、名称、副标题、菜单按钮和添加按钮，下方是搜索框和“会话 / 角色 / 群聊”分段按钮。
4. 移动端底部使用 Tab Bar，包含主要功能入口。当前选中项使用蓝紫色高亮。
5. 聊天页顶部显示返回按钮、角色头像、角色名、最新文字消息，以及电话、视频、更多按钮。
6. 聊天页底部输入栏固定，保持胶囊形态，发送按钮为圆形或圆角方形蓝紫按钮。
7. 移动端所有元素需要更紧凑，但不能拥挤，文字不能溢出，头像和按钮不能重叠。
8. 响应式断点建议：
   - 大屏桌面端：三栏布局。
   - 中等宽度：压缩侧栏或切换为简化布局。
   - 小屏移动端：列表页和聊天页分离显示。



实现方式要求：
1. 先阅读当前项目结构，找到现有 UI 组件、CSS 文件、页面入口和状态管理逻辑。
2. 在不破坏功能的前提下重构样式。优先通过 CSS 变量和组件 class 完成，不要大范围重写业务逻辑。
3. 建议建立统一 design tokens，例如：
   --color-bg;
   --color-surface;
   --color-surface-glass;
   --color-primary;
   --color-primary-2;
   --color-border;
   --color-text;
   --color-text-muted;
   --radius-sm;
   --radius-md;
   --radius-lg;
   --shadow-card;
   --shadow-floating;
4. 将按钮、输入框、卡片、头像、下拉菜单、弹窗、分段控件、消息气泡统一成可复用样式。
5. 所有交互状态都要有明确 hover / active / selected / disabled 样式。
6. 允许在不改变项目实现方式的前提下，直接引入外部 UI 框架，但不要引入负责和重型的 UI 框架。但应优先考虑使用原生 CSS / 现有技术栈实现。
7. 不要改变 API Key、Base URL、模型名称等设置项的字段含义。
8. 不要删除现有角色、现有聊天内容、现有按钮和已有入口。
9. 不要在界面中增加参考图没有要求的新模块、新文案或无关装饰。
10. 需要保证桌面端和移动端都能正常使用，不能只做静态展示图。



具体组件视觉要求：

搜索框：
- 白色或半透明白色背景。
- 细边框。
- 左侧搜索图标，右侧可以保留添加按钮。
- 高度使用 rem 或 clamp 控制，圆角使用响应式 design token。

分段按钮：
- 外层为浅色圆角容器。
- 当前选中项为蓝紫渐变背景，白色文字。
- 非选中项为白色或透明背景，灰色文字。

角色列表项：
- 高度使用 rem 或 clamp 控制。
- 头像左侧，名称和简介中间，时间/状态右侧。
- 当前选中角色使用极淡蓝紫背景。
- 列表项圆角使用响应式 design token。

聊天气泡：
- 左侧 AI：白色背景，细边框，柔和阴影。
- 右侧用户：浅蓝或蓝紫渐变背景，文字深色或白色，保持可读性。
- 时间文字使用浅灰色，小字号。
- 气泡之间保持足够纵向间距。
- 桌面端 max-width 可使用 min(60%, 42rem)，移动端 max-width 可使用 78% - 82%。

输入栏：
- 桌面端为底部悬浮长条，单行高度。
- 背景白色，边框浅灰，圆角使用响应式 design token。
- 图标左侧排列，输入文本区域居中，发送按钮右侧。
- 发送按钮使用蓝紫渐变和轻微阴影。
- 宽度跟随聊天区域自适应，不要写死固定宽度。

设置窗口：
- 整体为白色大圆角窗口。
- 桌面端左侧设置导航使用响应式宽度。
- 当前导航项使用淡蓝紫背景和左侧蓝紫指示。
- 表单输入框使用圆角、细边框、充足 padding。
- 保存按钮靠右下角，蓝紫渐变。
- 浮层宽度使用 min() 或 max-width 限制，避免在小屏幕溢出。

下拉菜单：
- 桌面端悬浮于触发按钮附近，移动端按需从触发按钮、屏幕底部或者以二级页面的方式浮出。
- 白色背景，圆角使用响应式 design token，阴影明显但柔和。
- 菜单项高度使用 rem 或 clamp 控制。
- 左侧图标，右侧文本。
- 危险项用红色文字或淡红背景。

人物资料卡片：
- 白色悬浮卡片，大圆角和柔和阴影。
- 左侧大头像，右侧角色名称、简介、标签。
- 标签使用浅色胶囊，例如“天才少女”“温柔体贴”“善解人意”“教学达人”。
- 底部两个按钮：主按钮“发起会话”为蓝紫渐变，次按钮“查看详情”为白色描边按钮。
- 卡片宽度使用 min()、max-width 和响应式留白，不能在小屏幕横向溢出。



验收标准：
1. 桌面端主聊天界面、设置内部窗口、下拉菜单、人物资料卡片都应完成。
2. 移动端列表页和聊天页必须分离展示，不能重叠。
3. 现有控件和功能入口全部保留。
4. 视觉上接近参考图：浅色背景、蓝紫主色、大圆角、柔和阴影、干净留白、现代聊天软件质感。
5. 文字、图标、头像、按钮在正常分辨率下清晰可见，不模糊、不挤压、不溢出。
6. 不允许只生成静态图片，必须落实到项目代码中。
7. 修改完成后，请说明改动了哪些文件、实现了哪些组件样式、移动端响应式如何生效。
